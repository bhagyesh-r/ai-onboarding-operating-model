# Repository Rules Template

## Scope

Describe which teams, folders, or workflows these rules cover.

## Always

- Require explicit task scope before significant edits
- Record verification evidence for every substantive change
- Escalate security-sensitive work for human review

## Never

- Commit secrets or private customer data
- Run destructive production actions without approval
- Hide automation side effects from reviewers

## Review gates

- Name the paths or task types that require specialist review
- Name any checks that must pass before merge
